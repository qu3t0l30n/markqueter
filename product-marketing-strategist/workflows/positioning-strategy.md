<positioning_strategy_workflow>

<objective>Define or sharpen a product's positioning — who it is for, what it replaces, why it is different, and why that difference matters. Grounded in customer discovery, competitive reality, and the Four Fits framework.</objective>

<required_reading>
- references/cagan-principles.md (Product Discovery, Four Risks, Vision/Strategy)
- references/reforge-growth.md (Four Fits Framework, Product Strategy Stack)
- references/anti-patterns.md (Positioning by Committee, Internal Echo Chamber, TAM Fantasy)
</required_reading>

<process>

**Phase 1: Discovery audit**

Before writing a single word of positioning, gather intelligence. Ask the user:

1. "Who are your best customers today — the ones who love you, renew, and expand?" (If they cannot answer specifically, flag this as a critical gap)
2. "What were these customers doing before they found you? What did they switch from?"
3. "In your last 5-10 won deals, what was the decisive reason they chose you?"
4. "In your last 5-10 lost deals, what was the decisive reason they did NOT choose you?"
5. "What is the one thing you do that is genuinely hard for competitors to replicate?"

If the user does not have answers to these, recommend conducting discovery first (Cagan's discovery techniques + Perri's Product Kata approach). Do not write positioning based on assumptions.

**Phase 2: Market category decision**

Using Reforge's Market-Product Fit framework, determine the market category strategy:

- **Existing category**: You compete directly with known alternatives. Positioning must differentiate clearly against the incumbent. Your messaging is about "better than X at Y."
- **Resegmented category**: You serve an underserved niche within an existing market. Positioning must define the niche and explain why generalist solutions fail for this audience. Your messaging is about "built specifically for [segment]."
- **New category**: You are creating something that did not exist before. Positioning must educate the market on the problem and the new way to solve it. Your messaging is about "the world has changed, and here is how to win in it."

Each strategy requires fundamentally different messaging — do not mix them.

**Phase 3: Build the positioning statement**

Using insights from Phase 1 and 2, construct the positioning using this framework:

```
FOR [target customer]
WHO [situation or problem they face]
[PRODUCT] IS A [market category]
THAT [key benefit / transformation enabled]
UNLIKE [primary alternative]
WE [key differentiator and why it matters]
```

Test the statement against these criteria:
- [ ] Is the target customer specific enough that you could find them in a room?
- [ ] Is the problem real and validated (not assumed)?
- [ ] Does the market category help or confuse the customer?
- [ ] Is the key benefit an outcome, not a feature?
- [ ] Is the differentiator defensible and honest?
- [ ] Would a prospect immediately understand why this matters to them?

**Phase 4: Build the proof architecture**

Positioning without proof is just a claim. For each element of the positioning, identify:

- **Customer evidence**: Quotes, case studies, metrics from real users
- **Product evidence**: Features, capabilities, or design decisions that prove the claim
- **Market evidence**: Industry trends, analyst quotes, or data that support the narrative
- **Competitive evidence**: Specific comparisons or win/loss data that validate differentiation

If proof is missing for any claim, flag it. Either find the proof or soften the claim.

**Phase 5: Competitive positioning matrix**

Map the competitive landscape:

| Dimension | Your Product | Competitor A | Competitor B | Alternative (status quo) |
|-----------|-------------|-------------|-------------|------------------------|
| Target user | | | | |
| Core problem solved | | | | |
| Key differentiator | | | | |
| Weakness / gap | | | | |
| Pricing model | | | | |
| Ideal use case | | | | |

Use this to identify:
- Your strongest competitive angles (where you clearly win)
- Your vulnerable flanks (where competitors are stronger)
- Positioning white space (unoccupied positions in the market)

**Phase 6: Messaging hierarchy**

Translate positioning into a layered messaging framework:

- **Level 1 — Headline**: The single most compelling statement (10 words or fewer)
- **Level 2 — Value proposition**: One paragraph that expands on the headline with specific value
- **Level 3 — Proof points**: 3-4 supporting pillars, each with evidence
- **Level 4 — Feature-benefit mapping**: Specific features tied to specific outcomes

Each level should work independently (a prospect who only sees Level 1 should understand your core value).

**Phase 7: Validation plan**

Positioning is a hypothesis until validated. Recommend specific validation steps:

1. Message testing with 5-10 target prospects (qualitative)
2. Landing page A/B test with different positioning angles (quantitative)
3. Win/loss interview analysis focused on messaging impact
4. Sales call recording review — does the positioning resonate in live conversations?

Output the positioning using the `templates/positioning-canvas.md` template.

</process>

<success_criteria>
- Positioning is grounded in customer evidence, not internal opinion
- Market category is explicitly chosen with rationale
- Positioning statement passes all six test criteria
- Proof architecture has no unsupported claims
- Competitive matrix reveals clear differentiation
- Messaging hierarchy works at every level
- Validation plan is concrete and actionable
</success_criteria>
